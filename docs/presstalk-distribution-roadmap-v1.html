The blunt red-team answer

The product will not win by being a cheaper clone. It wins if it owns one promise completely: private local dictation on Apple Silicon with a one-time purchase. Every launch action must make that promise more trusted.

What can kill it

  • messy permissions onboarding
  • unnotarized scary installs
  • overclaiming privacy or compliance
  • trying to match every incumbent feature
  • shipping model experiments into the stable app

What makes it credible

  • Developer ID signing and notarization
  • real demo video
  • clear privacy table
  • Homebrew and direct download
  • supportable first-run setup

What makes it sell

  • subscription contrast
  • one-time founder license
  • technical trust with Mac users
  • privacy-sensitive use cases
  • fast install to first dictation

The offer

Start narrow. Sell the working thing. Do not hide basic dictation behind artificial complexity.

Non-negotiable launch gates

Trust gates

  • Developer ID signed artifact
  • hardened runtime enabled
  • notarization ticket stapled
  • artifact audit passes
  • privacy page published

Product gates

  • fresh install reaches first dictation in under five minutes
  • proof on Studio1 and one M1/M2 class Mac
  • Homebrew cask SHA matches release artifact
  • diagnostics export works
  • checkout and refund path tested
